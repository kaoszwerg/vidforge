# `.claude/memory/` — repo-resident context for agents

These notes are committed to the repo and survive a fresh clone. Per ADR-CORE-003 the repository is the
single source of truth, and this directory is symlinked into the per-user Claude memory location by
`scripts/setup-claude-memory.sh` (run from the `SessionStart:new` hook) — so local and repo memory are
the same files.

## What goes where

- **Behavioural rules** → `.claude/rules/*.md` (not here)
- **Architectural decisions** → `docs/adr/*.md` (not here)
- **Top-level facts / conventions** → `CLAUDE.md` (not here)
- **Progress / implementation state** → `PLAN.md` (not here)
- **Durable project context that fits nowhere above** → this directory

## Format

Every entry (except this `README.md` and the generated `MEMORY.md`) carries front-matter:

```yaml
---
id: mem:project-scope
title: …
tldr: "≤160 chars"
scope: project # project | governance | data | frontend | backend
load: core | conditional | archival
type: project # project | feedback | reference | user
---
```

Entries of `type: feedback` or `type: project` must additionally contain `**Why:**` and
`**How to apply:**` lines. `scripts/lint-memory.mjs` enforces this and that `MEMORY.md` links every entry
with no orphans or dead links. `MEMORY.md` is generated by `scripts/sync-index.mjs` — do not hand-edit it.
